
NODE.JS BANKING SYSTEM BACKEND - MINI PROJECT DOCUMENTATION
===========================================================

1. PROJECT OVERVIEW
-------------------
This project is a simple Banking System backend built using Node.js and Express.js.
It simulates basic banking operations such as account creation, deposit, withdrawal, and transfer between accounts.

The system is designed as a learning project to understand backend development, APIs, authentication, and database design.

2. TECH STACK
-------------
- Runtime: Node.js
- Framework: Express.js
- Database: MongoDB (Mongoose ORM)
- Authentication: JWT (JSON Web Token)
- Password Security: bcrypt
- Tools: Postman for API testing

3. CORE MODULES
--------------
1. Authentication Module
   - User registration
   - User login
   - JWT token generation

2. User Module
   - User profile management
   - View account details

3. Account Module
   - Create bank account
   - Check balance
   - Account status (active/inactive)

4. Transaction Module
   - Deposit money
   - Withdraw money
   - Transfer money between accounts
   - Transaction history

5. Admin Module (optional)
   - View all users
   - Monitor transactions

4. DATABASE DESIGN
------------------

User Collection:
- _id
- name
- email
- password (hashed)
- createdAt

Account Collection:
- _id
- userId (reference to User)
- accountNumber
- balance
- status (active/inactive)
- createdAt

Transaction Collection:
- _id
- fromAccount
- toAccount
- amount
- type (deposit/withdraw/transfer)
- timestamp

5. API ENDPOINTS
---------------

Auth Routes:
- POST /api/auth/register -> Register user
- POST /api/auth/login -> Login user

User Routes:
- GET /api/user/profile -> Get user profile

Account Routes:
- POST /api/account/create -> Create account
- GET /api/account/balance -> Get balance

Transaction Routes:
- POST /api/transaction/deposit -> Deposit money
- POST /api/transaction/withdraw -> Withdraw money
- POST /api/transaction/transfer -> Transfer money
- GET /api/transaction/history -> Get transaction history

6. APPLICATION FLOW
-------------------

1. User registers using email and password
2. Password is hashed using bcrypt before storing
3. User logs in and receives JWT token
4. Token is used for all protected routes
5. User creates a bank account
6. User performs transactions:
   - Deposit increases balance
   - Withdraw decreases balance (with validation)
   - Transfer moves money between accounts
7. Every transaction is stored in the database

7. SECURITY FEATURES
-------------------
- Password hashing using bcrypt
- JWT authentication for protected routes
- Input validation for all requests
- Basic rate limiting (optional)
- Secure environment variables (.env)

8. VALIDATION RULES
------------------
- Email must be valid format
- Password must be minimum length (e.g., 6-8 characters)
- Balance cannot go below zero
- Transfer requires sufficient balance

9. ERROR HANDLING
-----------------
- Standard API error responses:
  { success: false, message: "Error message" }

- Proper status codes:
  200 - Success
  201 - Created
  400 - Bad Request
  401 - Unauthorized
  404 - Not Found
  500 - Server Error

----------
This project demonstrates a simple but realistic banking backend system.
